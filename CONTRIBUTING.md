# Contributing

Thanks for your interest in contributing! Please take a moment to review this guide.

## Reporting Bugs

- Search [existing issues](../../issues) before opening a new one
- Include steps to reproduce the problem
- Include what you expected to happen vs what actually happened

## Suggesting Features

- Open an [issue](../../issues) to discuss the idea first
- Explain the use case and why it would be useful

## Pull Requests

- One PR per fix or feature
- Target the `main` branch
- Follow the existing code style
- Keep changes focused and minimal

## Security

Do **not** file security issues publicly. See [SECURITY.md](SECURITY.md) for how to report vulnerabilities privately.
